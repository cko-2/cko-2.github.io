## Invisible instruction manual

Five minutes ago, I had no idea what a design pattern was. Tasked to write this essay, I did a bit of research and found that it was simply a label to what I had been doing throughout the course. The term "design patterns" was invisible, yet present the entire time. While design patterns can seem complex with the many names for each principle and pattern, to put it simply, design patterns are, design, patterns. They are the processes we go through when creating software. 

## Using design patterns without knowing

For our final project, my group and I had utilized many design patterns without knowing. We were building a webapp for the UH ITS department that would allow student help desk workers to contribute to a shared library of email templates. This would allow for more efficient workflows and uniformity among the many employees. Within each template, it would append a greeting and signature, which was the first design pattern: the Template Method Pattern. Using a form that would serve as the skeleton allowed for customizable templates and used this design pattern perfectly. Within the rest of my project I noticed more design patterns such as the Component Pattern with React components, Observer Pattern with useState, and Iterator Pattern with .map(). It seemed that I was using these design patterns everywhere in my project, and it now provides structure and organization now that I can recognize these.
